# פרויקט אמצע — Dashboard לסקר החברתי

Dashboard אינטראקטיבי לניתוח **הידע הפיננסי והמוכנות לפרישה** על בסיס הסקר החברתי של הלמ"ס 2024 (6,907 משיבים).

**האתר החי:** https://orarr2.github.io/Mid_project_Social_Survey/

## מה יש בריפו הזה

הריפו כולל את **שכבת ה-Deployment** של הפרויקט (החלק שלי בעבודת הצוות). מה שנכלל:

- עיצוב ובנייה של דף Dashboard עצמאי (`index.html`)
- שאלון בן 10 שאלות על ידע פיננסי, שחוסם את הכניסה ל-Dashboard — לפני שכל השאלות נענו, שום ויזואליזציה לא מוצגת
- ניתוח ויזואלי אינטראקטיבי של הדאטה הגולמי (סקירה, דמוגרפיה, התנהגות פיננסית, מוכנות לפרישה, פוקוס דור צעיר, פרופיל אישי מול האוכלוסייה)
- טאב Model עם Placeholder ממתין למודל המאומן שיגיע משלב המידול ב-pipeline

ניקוי דאטה, imputation, feature engineering ואימון מודל מטופלים בשלבים אחרים של ה-pipeline של הצוות (שי ועובד) ולא כלולים בריפו הזה.

## האתר החי

Dashboard מפורסם כאתר סטטי ב-GitHub Pages ישירות מ-branch `main`:

- כתובת: https://orarr2.github.io/Mid_project_Social_Survey/
- שום דבר לא צריך להיות מותקן אצל הקורא. הכל (Plotly.js, המפה, תמונת ענן המילים) מוטמע בתוך `index.html`.

## חלוקת עבודה בצוות

ראה את התמונה [`pipeline.png`](pipeline.png) בריפו הזה — לוח שלוש שבועות עם:

- **שי** — ניקוי דאטה + בחירת פיצ'רים (מוסר `data_24_clean.parquet`)
- **עובד** — אימון מודל + SHAP (מוסר `xgb_model.pkl`)
- **אור (אני)** — Dashboard + Deployment (מוסר `index.html` על Pages)
- **בני** — מצגת + Retrain (מוסר מצגת סופית + refit)

פגישת סטטוס בכל יום שישי בשעה 17:30. ייעוץ עם המרצה ב-17.07. הגשה סופית ב-29.07.

## איך מריצים מקומית

קובץ הדאטה של הסקר לא נכנס לריפו. יש למקם אותו מקומית ב:

```
data/Social_Survey/data_24.xlsx
```

ואז:

```
pip install -r requirements.txt
jupyter notebook "Mid Project Dashboard.ipynb"
# Run > Run All Cells
```

המחברת כותבת `index.html` לצדה ופותחת אותו בדפדפן. הריצה הראשונה קוראת את קובץ ה-Excel (כדקה) וכותבת parquet cache לריצות עתידיות מהירות.

## שלבי ה-pipeline

1. איסוף וניקוי דאטה — שלב מוקדם (שי). מוציא dataset נקי.
2. Feature engineering ואימון מודל — שלב מוקדם (עובד). מוציא artifact של XGBoost + SHAP.
3. **Deployment ו-Dashboard (הריפו הזה).** צורך את הדאטה הגולמי היום, וייצרוך בעתיד גם את פלט המודל המאומן כדי למלא את טאב ה-Model.
