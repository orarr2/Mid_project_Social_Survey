# פרויקט אמצע - Dashboard לסקר החברתי

Dashboard אינטראקטיבי לניתוח **הידע הפיננסי והמוכנות לפרישה** על בסיס הסקר החברתי של הלמ"ס 2024 (6,907 משיבים).

## ויזואליזציות ב-Dashboard

הדאטה בשימוש **כמו שהוא** - אין הסרת ערכים חסרים, אין הסרה של שורות, אין החלפה של קודים שמורים. קודי הלמ"ס `888888` (לא יודע / מסרב) ו-`999999` (לא רלוונטי / סינון) מופיעים בגרפים כקטגוריות בפני עצמן, היכן שהם קיימים.

## איך מריצים מקומית

קובץ הדאטה של הסקר לא נכנס לריפו. יש למקם אותו מקומית ב:

```
data/Social_Survey/data_24.xlsx
```

ואז:

```
pip install -r requirements.txt
jupyter notebook "Mid Project Dashboard.ipynb"
# Run > Run All Cells
```

המחברת כותבת `index.html` לצידו ופותחת אותו בדפדפן. הריצה הראשונה קוראת את קובץ ה-Excel (כדקה) וכותבת parquet cache לריצות עתידיות מהירות.

## מבנה הפרויקט

```
Mid Project Dashboard.ipynb   Pipeline הבנייה: קורא את הסקר, בונה את הגרפים,
                              מייצר את index.html + מרים שרת מקומי
index.html                    דף Dashboard עצמאי (מפורסם ב-GitHub Pages)
requirements.txt              תלויות Python
README.md                     קובץ README באנגלית
README_he.md                  קובץ README בעברית (הקובץ הזה)
NOTEBOOK_WALKTHROUGH_he.md    הסבר תא-אחר-תא של המחברת (לפגישת הזום)
```

## שלבי ה-pipeline

1. איסוף וניקוי דאטה - שלב מוקדם (שי). מייצר dataset נקי.
2. Feature engineering ואימון מודל - שלב מוקדם (עובד). מוציא artifact של XGBoost + SHAP.
3. **Deployment ו-Dashboard (הריפו הזה).** צורך את הדאטה הגולמי היום, ובעתיד יצרוך גם את פלט המודל המאומן כדי למלא את טאב ה-Model.
